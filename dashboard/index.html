Consistency. Same input rendered 10 times
Does the model produce the same audio twice for the same prompt? We rendered the same 12-word sentence 10 times and measured two things: total audio duration, and root-mean-square (RMS) loudness. Both are reported as coefficient of variation (CV): the standard deviation divided by the mean.

How to read CV. A CV of 0.029 means the run-to-run variation is about 2.9% of the mean. CV below 0.05 is near-identical output across runs (essentially deterministic). CV between 0.05 and 0.15 is mild variability. Higher than 0.15 means the model is rendering the same input visibly differently each time.

In our test, duration is very stable (CV 0.029). The model decides the same pacing every time. But loudness varies ±11% (CV 0.107). Delivery intensity is not fully deterministic. If you need identical volume across cached prompts, you'd want to normalize loudness in post-processing.
